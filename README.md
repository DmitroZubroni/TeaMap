# 茶 · Атлас чая

Интерактивная карта чаёв мира: React + Vite + Tailwind + MapLibre GL.
Карта — настоящая география (страны, реки, озёра), перекрашенная под
собственную палитру поверх бесплатного векторного стиля CARTO Positron
(ключ API не нужен).

## Как это устроено

- **Три уровня зума**, переключаются автоматически по текущему zoom карты:
  1. **Мир** (zoom ≤ 3.2) — пины стран-производителей чая. Страна с данными
     (сейчас только Китай) выделена, остальные показаны затемнённой меткой
     «скоро».
  2. **Страна** (3.2 < zoom ≤ 6.4) — пины регионов/гор (`regions/{country}.json`).
  3. **Регион** (zoom > 6.4) — отдельные точки чая (`teas/{country}.json`),
     цвет точки = цвет настоя категории (см. легенду снизу слева).
- Клик по точке чая грузит полную карточку `tea-details/{country}/{id}.json`
  и открывает боковую панель (снизу — на мобильных) с характеристиками,
  описанием, историей, похожими чаями и источниками.
- Все данные читаются в рантайме через `fetch('/tea-data/...')`, а не через
  `import` — см. `src/lib/api.js`. Это значит: обновил JSON в
  `public/tea-data/` → обновление видно сразу, без пересборки фронтенда.

## Структура

```
src/
  lib/
    api.js            — fetch-обёртки над /tea-data с кэшем
    categoryStyle.js   — палитра категорий (цвет = цвет настоя)
    countryMeta.js      — центроиды стран для мирового вида
    markers.js          — DOM-фабрики пинов (страна/регион/чай)
    mapStyle.js          — перекраска CARTO Positron под палитру атласа
  components/
    MapView.jsx     — карта, зум-стейджинг, все три слоя пинов
    Breadcrumb.jsx     — заголовок + хлебные крошки навигации
    Legend.jsx           — легенда категорий
    TeaPanel.jsx          — карточка чая
    Toast.jsx              — уведомление «данные скоро появятся»
public/
  tea-data/    — содержимое вашего репозитория с данными (см. ниже)
```

## Как подключить данные из TeaMenuFoundation

Этот проект уже содержит копию присланного `tea-data.zip` в
`public/tea-data/`. Если структура в репозитории `TeaMenuFoundation` будет
обновляться — просто синхронизируйте содержимое `tea-data/` из репозитория
в `public/tea-data/` этого проекта (структура папок один в один). Менять
код не нужно, пока сохраняются:
- имена файлов `teas/{country}.json`, `regions/{country}.json`,
  `tea-details/{country}/{id}.json`, `countries.json`, `categories.json`,
  `sources.json`;
- поля `lat`/`lng` в индексах и `id`, совпадающий с именем файла карточки.

Если решите объединить оба репозитория в один — можно вынести
`public/tea-data` в git submodule, указывающий на `TeaMenuFoundation`, или
просто держать данные как обычную папку в этом же репозитории.

## Запуск

```bash
npm install
npm run dev       # http://localhost:5173
npm run build      # прод-сборка в dist/
npm run preview     # локальный просмотр prod-сборки
```

## Известные ограничения текущей версии (MVP)

- Реализованы только страны/регионы/чаи Китая — остальные страны в
  `countries.json` показываются как метки «скоро» на мировом виде.
- Нет поиска и фильтров по категориям — по договорённости, это следующая
  итерация.
- Изображений в базе нет (`gallery` пустой), поэтому карточка чая пока
  текстовая.
- Привязка чая к региону на уровне зума «регион» сделана через координаты
  самого чая, а не строгое сопоставление `region` в `teas/*.json` с
  `regions/*.json` — при желании ужесточить фильтрацию, `MapView.jsx`
  (эффект «tea-stage pins») — то место, где это нужно поменять.
