# tea-data — открытая база данных мирового чая

JSON-база чаёв мира с географической привязкой для интерактивной чайной карты.

## Структура репозитория

```
tea-data/
├── countries.json          # справочник стран
├── categories.json         # справочник категорий чая
├── sources.json            # справочник источников информации
│
├── teas/                   # ЛЁГКИЕ индексы для карты, один файл на страну
│   ├── china.json          # только id, координаты, категория, регион
│   ├── india.json          # (будет добавлен)
│   └── ...
│
├── tea-details/             # ПОЛНЫЕ карточки чая, шардировано по странам
│   ├── china/
│   │   ├── da_hong_pao.json
│   │   └── ...
│   ├── india/               # (будет добавлен)
│   └── ...
│
├── regions/                 # точки регионов/гор для отображения на карте
│   ├── china.json
│   └── ...
│
├── geojson/                 # границы регионов (опционально, для полигонов)
│
└── images/                  # изображения (не хранятся в git — см. ниже)
    ├── teas/
    ├── regions/
    ├── mountains/
    └── processing/
```

## Принцип загрузки данных (двухуровневый)

1. **Карта грузит только `teas/{country}.json`** для стран, попадающих в текущий viewport/zoom — лёгкий файл с координатами, категорией и id.
2. **При клике на точку** фронтенд догружает полную карточку из `tea-details/{country}/{id}.json`.

Это даёт быструю первую загрузку карты и не требует тянуть всю базу целиком.

## Правила добавления нового чая

1. Создать файл `tea-details/{country}/{id}.json` по схеме ниже.
2. Добавить облегчённую запись в `teas/{country}.json` (id, name, country, category, lat, lng, region).
3. Если регион новый — добавить точку в `regions/{country}.json`.
4. Обновить `teaCount` в `countries.json` (опционально, можно автоматизировать скриптом).

## Схема карточки (`tea-details/{country}/{id}.json`)

```json
{
  "id": "string, уникальный slug",
  "name": "string, название на русском",
  "aliases": ["альтернативные названия, латиницей и оригинальным письмом"],
  "country": "id страны из countries.json",
  "category": "id категории из categories.json",
  "location": {
    "province": "", "city": "", "region": "", "mountain": "",
    "latitude": 0.0, "longitude": 0.0, "altitude": "строка-диапазон"
  },
  "characteristics": {
    "cultivar": "", "oxidation": "", "roast": "", "harvestSeason": "",
    "waterTemperature": "", "steeps": "", "leafWeight": "", "teaware": "",
    "rarity": 1, "difficulty": 1, "agesWell": true
  },
  "description": {
    "short": "", "overview": "", "leaf": "", "liquor": "",
    "aroma": "", "taste": "", "aftertaste": "", "processing": ""
  },
  "history": {
    "origin": "", "development": "", "modern": "",
    "interestingFacts": ["массив фактов"]
  },
  "relatedTeas": ["id других чаёв из базы"],
  "gallery": ["пути/URL к изображениям"],
  "sources": ["id источников из sources.json"]
}
```

## Правила добавления новой страны

1. Добавить запись в `countries.json`.
2. Создать `teas/{country}.json` (пустой массив `[]` для старта).
3. Создать `tea-details/{country}/` (папку можно создать вместе с первым файлом).
4. Создать `regions/{country}.json`.

## Масштабирование (ориентиры)

| Объём базы | Рекомендация |
|---|---|
| до ~300 карточек | текущая структура — без изменений |
| ~300–1000 | добавить build-скрипт, генерирующий `teas/*.json` автоматически из `tea-details/**/*.json` (единый источник правды для координат) |
| 1000+ | рассмотреть переход раздачи через реальный API вместо статичных файлов |

## Источники

Данные собираются из нескольких источников (см. `sources.json`) и переписываются своими словами, а не копируются дословно.
